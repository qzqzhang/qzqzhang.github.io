如何搭建个人博客网站

前言

本文将记录一下如何从零开始，将Hexo项目部署到github pages和服务器（Ubuntu 20.04）上。（笔者所用服务器环境为云服务器，本地机为windows11）。

正文

一、将hexo部署到github pages上

1、登录自己的GitHub并创建一个仓库

注：如果你希望你的站点能通过 <你的 GitHub 用户名>.github.io 域名访问，你的 repository 应该直接命名为 <你的 GitHub 用户名>.github.io。

在此笔者不做过多介绍了。

2、安装Git

Git是开源的分布式版本控制系统，用于敏捷高效地处理项目。我们的博客网站在本地搭建好了需要使用Git同步到GitHub上。或者使用full版的Cmder，里面自带了Git for Windows。

Git官网下载地址：https://git-scm.com/download

安装成功后将Git与Github账号绑定，鼠标右键打开Git Bash，或者在菜单栏中搜索

接着设置user.name和user.email配置信息：

在你的本地机中找到生成的.ssh文件夹中的id_rsa.pub密钥文件，将其中内容复制。

打开GitHub_Settings_keys页面，新建new SSH Key，标题随意，将刚刚复制的id_rsa.pub内容粘贴进去，点击添加。

可以在Git Bash或者Cmder中测试看公钥是否设置成功。

如上则设置成功。

3、安装Node.js

Hexo基于Node.js，Node.js下载地址：Download | Node.js ,安装完成后可在命令行输入node -v检测，此处为了使用npm命令。

到此安装hexo用到的环境完成。

Hexo是一个个人博客框架，可以根据自己喜好选择要下载的主题，这边需要创建一个文件夹，hexo框架和后续发布的网页都存在这个文件夹中。创建好后，进入该文件夹根目录，打开cmder/cmd，使用npm命令安装Hexo。（此处命令皆在新建的这个文件夹中进行）

本地完成后需要推送到网页上，在站点根目录下能看到_config.yml文件，它为站点的配置文件，发布到GitHub pages或服务器上需要对它进行修改。

笔者此处所选主题为fluid，主题自行下载（默认放到node_modules文件夹内），下载后主题也有个配置文件，为_config.fluid.yml，进入主题目录中可以将它复制到站点目录下（该文件可以对网页所显示进行修改）。
